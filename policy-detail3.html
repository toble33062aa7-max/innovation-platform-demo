《遠距醫療照護資料交換標準》草案

草案摘要

本標準草案旨在定義一套統一的資料格式與應用程式介面（API）規範，以促進不同遠距醫療服務平台、電子病歷（EMR）系統及個人健康紀錄（PHR）應用之間的互操作性。目標是讓患者的健康數據能夠在安全、可靠的前提下，於不同醫療照護提供者之間順暢流動，實現以患者為中心的整合照護。

主要規範內容包含：

  • 生理數據（如血壓、血糖、心率）的標準格式。
  • 視訊問診紀錄的結構化儲存建議。
  • 基於 HL7 FHIR 標準的 API 規範。
  • 身份驗證與資料加密的安全要求。

為何需要您的意見？

新創團隊是推動數位健康創新的前線。您的實務經驗對於確保本標準的**技術可行性**、**市場適用性**及**未來擴展性**至關重要。我們特別希望傾聽您在以下方面的觀點：

  • 導入本標準對現有產品架構的技術挑戰與預估成本。
  • 標準中定義的資料欄位是否完整，能否滿足創新應用的需求？
  • API 規範的設計是否友善開發者，是否存在潛在的實作障礙？
  • 對於資料安全與隱私保護，是否有更佳的實務建議？

意見提交指南

為利於意見彙整，請下載「意見回饋範本」，依格式填寫後，透過線上表單上傳提交。若有其他補充文件，亦可一併上傳。

  1. 第一步：從右側下載「草案全文」及「意見回饋範本」。
  2. 第二步：詳閱草案內容，並將您的建議填寫於範本檔案中。
  3. 第三步：點擊「前往線上意見提交表單」，填寫基本資料並上傳已完成的範本檔案。
  4. 第四步：於截止日期 2024-11-30 前完成提交。
